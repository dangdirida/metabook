## 1. Executive Summary

MetaBook는 김영사 및 주니어김영사가 출판해 온 전체 도서 카탈로그를 디지털 인터랙티브 경험으로 전환하는 통합 독서 플랫폼입니다. 독자는 실물 책에 인쇄된 QR 코드를 통해 입장하며, 책의 캐릭터·저자와 AI 대화를 나누고, 삽화 속 공간으로 직접 들어가며, 다른 독자들과 커뮤니티를 형성합니다.

### 핵심 가치 제안 (Value Proposition)

| # | 가치 |
|---|------|
| 📖 | 모든 책 한 권 = 하나의 살아있는 세계 |
| 🤖 | AI Agent: 주인공·저자·등장인물과 1:1 대화 |
| 🌐 | WorldLabs Marvel 기반 삽화 3D 공간 탐험 |
| 🎨 | 독자 2차 창작 → OGQ Market 직접 업로드·판매 |
| 👥 | 책별 커뮤니티 채팅방 + 한정판 굿즈 커머스 |

### 1.1 프로젝트 배경

디지털 네이티브 독자층(MZ·알파세대)은 단방향 콘텐츠 소비를 넘어 참여·상호작용을 원합니다. 기존 전자책은 텍스트 디지털화에 그쳤으나, MetaBook는 각 책을 "IP 유니버스"로 확장하여 독자가 능동적 참여자가 되도록 합니다.

### 1.2 성공 지표 (Success Metrics)

| 지표 | 목표 (론칭 후 6개월) | 비고 |
|------|---------------------|------|
| 월간 활성 사용자(MAU) | 50,000명 | QR 진입 포함 |
| AI Agent 대화 세션/월 | 200,000건 | 책 당 평균 20건/일 |
| 커뮤니티 채팅 메시지/월 | 500,000건 | - |
| 2차 창작 OGQ 업로드 | 1,000건 | 승인 기준 |
| QR 코드 → 플랫폼 전환율 | 35% | 실물 책 대비 |
| 평균 세션 시간 | 12분 이상 | 일반 뉴스 앱 대비 3× |

## 2. 사용자 정의 및 페르소나

### 2.1 사용자 유형

| 유형 | 설명 | 핵심 니즈 | 진입 경로 |
|------|------|----------|----------|
| 일반 독자 | 책을 구매한 기존 독자 | AI 대화, 깊이 있는 독서 | QR 코드 스캔 |
| 팬덤 독자 | 시리즈 완독자, 커뮤니티 활동가 | 커뮤니티, 2차 창작 | QR 또는 직접 접속 |
| 크리에이터 | 2차 창작 제작자 | OGQ 업로드, 수익화 | 플랫폼 → OGQ |
| 학부모/교사 | 주니어김영사 독자 보호자 | 교육 콘텐츠, 안전한 환경 | QR 코드 스캔 |
| 어드민 | 김영사 내부 운영자 | 책 관리, Agent 설정 | 어드민 포털 |

### 2.2 핵심 페르소나

#### 페르소나 A: "독서덕후" 이지수 (25세)

서울 거주 직장인. 김영사 도서 정기 구독. 주인공에게 이입하는 몰입형 독서 스타일.

**Goals**
- 책 속 세계관 깊이 탐구
- 주인공과 직접 대화하며 "if" 시나리오 경험
- 비슷한 취향의 독자 커뮤니티 형성

**Pain Points**
- 다 읽은 뒤 책 세계가 끝나는 아쉬움
- 온라인에 분산된 팬 커뮤니티

---

#### 페르소나 B: "창작자" 박민준 (19세)

대학생. 일러스트 취미. OGQ 스티커 1종 판매 경험. 좋아하는 소설 2차 창작 희망.

**Goals**
- 공식 IP 기반 2차 창작 합법적으로 제작
- OGQ Market 업로드로 수익 창출
- 창작 결과물 독자 커뮤니티에 공유

**Pain Points**
- IP 저작권 불명확 → 창작 포기
- 업로드 플랫폼 접근성 부족

---

#### 페르소나 C: "주니어 엄마" 김서연 (38세)

초등 2학년 자녀 보유. 주니어김영사 과학책 구매. 아이 독서 습관 형성 관심.

**Goals**
- 아이가 책을 재미있게 읽도록 돕기
- 교육적인 AI 대화 경험 제공
- 안전한 어린이 전용 환경

**Pain Points**
- 성인 콘텐츠 노출 걱정
- 앱 설치 없이 바로 접근 원함

---

#### 페르소나 D: "운영자" 최다은 (32세)

김영사 디지털콘텐츠팀. MetaBook 전체 책 관리 담당.

**Goals**
- AI Agent 캐릭터 설정·학습 데이터 관리
- 2차 창작 승인·거절 워크플로
- 사용자 행동 데이터 분석

**Pain Points**
- 책마다 개별 관리 툴이 달라 비효율
- AI 응답 품질 모니터링 어려움

## 3. 정보 아키텍처 (IA) 및 네비게이션

### 3.1 전체 사이트 맵

```
/
├── /auth                    (구글 로그인)
├── /library                 (전체 도서 목록 — 랜딩)
│   ├── /library/{bookId}    (개별 책 3패널 메인)
│   │   ├── /world/{imageId} (월드 모델 3D 공간)
│   │   ├── /agent/{agentId} (AI Agent 대화 전용)
│   │   └── /community       (책 커뮤니티)
├── /creations               (2차 창작 허브)
│   └── /creations/{type}    (유형별 필터)
├── /shop                    (굿즈 커머스)
└── /admin                   (어드민 포털)
    ├── /admin/books
    ├── /admin/agents
    ├── /admin/creations
    └── /admin/analytics
```

### 3.2 URL 구조 및 QR 코드 매핑

| QR 위치 | URL 패턴 | 랜딩 화면 |
|---------|---------|----------|
| 책 표지/뒷면 전체 QR | `/library/{bookId}` | 해당 책 3패널 메인 |
| 특정 삽화 옆 QR | `/world/{bookId}/{imageId}` | 해당 이미지 월드 공간 |
| 챕터 시작 QR | `/library/{bookId}?chapter={n}` | 해당 챕터 포커스 |
| 굿즈 패키지 QR | `/shop/{bookId}` | 해당 책 굿즈 스토어 |

## 4. 기능 명세 (Feature Specifications)

### 4.1 인증 및 온보딩

#### 4.1.1 구글 로그인 (Google OAuth 2.0)

- 구글 계정 1클릭 로그인
- 연령 인증: 14세 미만은 보호자 동의 플로우 (주니어 모드 자동 활성화)
- 첫 로그인 시 3단계 온보딩: 관심 장르 선택 → 닉네임 설정 → 추천 책 3권 표시
- QR 진입 시: 로그인 전이면 소셜 로그인 팝업 → 완료 후 원래 목적지 복귀

---

### 4.2 도서 목록 페이지 (`/library`)

#### 4.2.1 레이아웃

그리드 형태의 도서 카드 목록. 필터/정렬 상단 바.

- **카드 요소**: 표지 이미지, 제목, 저자, 출판사(김영사/주니어김영사), 장르 태그, AI Agent 수, 커뮤니티 인원수
- **정렬**: 최신순 / 인기순(커뮤니티 활성) / 2차 창작 많은 순
- **필터**: 출판사, 장르, 연령대(어린이/청소년/성인), 언어
- **검색**: 제목, 저자, 캐릭터명 통합 검색
- **반응형**: 데스크탑 5열 / 태블릿 3열 / 모바일 2열

---

### 4.3 책 상세 페이지 — 3패널 레이아웃 (`/library/{bookId}`)

```
[ 왼쪽 패널 (280px) ] | [ 가운데 패널 (flex) ] | [ 오른쪽 패널 (380px) ]
  책 목록 + 이미지 탐색  |   책 본문 + AI 대화    |   커뮤니티 채팅 + 창작
```

---

#### 4.3.1 왼쪽 패널: 도서 탐색 네비게이션

**책 목록 섹션**

- 상단: 현재 보고 있는 책 표지 + 제목
- "내 서재" 탭: 사용자가 QR 진입했거나 북마크한 책 목록
- "전체 도서" 탭: 전체 도서 미니 목록 (스크롤 가능)
- 각 책 항목 클릭 시 해당 책 3패널로 전환

**이미지 탐색 아코디언**

- 책 제목 우측의 [펼치기 ▼] 버튼 클릭 시 해당 책의 삽화/사진 썸네일 목록 펼침
- 썸네일 격자: 2열 × N행
- 각 썸네일 호버 시: 해당 장면 미리보기 툴팁
- 썸네일 클릭 시: 월드 모델 뷰어(`/world/{bookId}/{imageId}`)로 이동 (새 탭)
- 썸네일에 별도 QR 아이콘 표시 → 클릭 시 QR 코드 팝업 (실물 공유용)

---

#### 4.3.2 가운데 패널: 책 본문 + 인터랙션

가운데 패널의 인터랙션은 **점진적 노출(Progressive Disclosure)** 원칙을 따른다. 사용자에게 한 번에 모든 기능을 설명하지 않고, 시각적 단서로 자연스럽게 발견하게 유도한다.

**본문 표시**

- 책 텍스트 렌더링: 타이포그래피 최적화, 야간 모드 지원
- 챕터/페이지 네비게이션 (상단 프로그레스 바)
- 폰트 크기·줄간격 조절 (사용자 설정 저장)

**첫 방문 온보딩 힌트 (1회만 노출)**

- 첫 진입 시 오버레이 힌트 1개만 표시: `"인물 이름을 눌러보세요 👆"`
- 이미지·텍스트 인터랙션은 힌트에서 언급하지 않음 — 시각적 단서로만 유도
- 힌트는 3초 후 자동 사라짐 또는 화면 탭 시 즉시 닫힘
- 이후 방문부터는 힌트 미표시 (로컬스토리지 플래그)

**인터랙션 A: 인물 클릭 (주 진입점)**

- 등장인물 이름을 본문 내에서 색상(primary-500 계열)으로 상시 하이라이트 처리
- 색상만으로 "클릭 가능"임을 암시 — 별도 아이콘이나 설명 불필요
- 인물명 클릭 → 우측 패널에 해당 AI Agent 채팅창이 열림
- 인물 정보 팝업 (이름, 역할, 등장 챕터 요약)

**인터랙션 B: 이미지 클릭 (시각적 배지로 암시)**

- 삽화/사진 우하단에 작은 🌐 배지 항상 표시 → "들어갈 수 있다"는 시각적 단서
- 이미지 클릭 → `"월드 공간으로 들어가기"` 확인 팝업
- 확인 시 WorldLabs Marvel 뷰어 새 창/탭으로 오픈
- 3D 공간 내에서 등장인물 AI Agent와 대화 가능

**인터랙션 C: 텍스트 선택 (가르치지 않는 인터랙션)**

- 별도 안내 없음 — 사용자가 텍스트를 드래그/롱프레스하면 자연스럽게 메뉴 등장
- 모바일 사용자는 이미 텍스트 선택 동작에 익숙 → 학습 비용 0
- 선택 시 컨텍스트 메뉴: 하이라이트 / 메모 추가 / 이 부분으로 2차 창작 / AI에게 질문 / 공유
- "이 부분으로 2차 창작" 선택 시: 2차 창작 에디터로 선택 텍스트 전달

---

#### 4.3.3 오른쪽 패널: 커뮤니티 + AI Agent 채팅

**상단 탭 구성**

- `[AI 대화]` 탭 — 기본 선택
- `[커뮤니티]` 탭
- `[2차 창작]` 탭

**[AI 대화] 탭**

- 채팅방 목록: 주인공, 저자, 주요 조연 등 Agent 목록
- 각 채팅방 상단: 책 제목 + Agent 이름 + 아바타
- 대화 UI: 말풍선 형태, 타이핑 애니메이션
- Agent 전환: 현재 채팅 중 다른 인물로 빠른 전환 가능 (토글)
- 대화 기록: 사용자별 영구 저장, 이어서 대화 가능
- 공유하기: 대화 스냅샷을 이미지로 저장 → SNS 공유

**[커뮤니티] 탭**

- 이 책에 관심 있는 사용자들의 실시간 단체 채팅방
- 입장 조건: 로그인 필수, 욕설 필터 적용
- 채팅 멤버 목록: 우측 상단 인원 수 표시 → 클릭 시 현재 활성 유저 리스트
- `[+]` 버튼 → "초대하기": 링크 복사 or 카카오·라인 공유 (본인 포함 최대 2명 추가 초대 — 책 구매자 기준)
- 메시지 유형: 텍스트, 이모지, 이미지, 2차 창작 결과물 링크 첨부
- 관리자 공지 고정 메시지 (상단)

**[2차 창작] 탭**

- 이 책의 승인된 2차 창작물 갤러리
- 정렬: 랭킹순 / 최신순 / 좋아요순
- `[내 창작물 올리기]` 버튼 → 2차 창작 업로드 플로우
- 각 창작물 클릭 → 새 창으로 상세 뷰
- `[OGQ Market에서 구매]` 버튼 → OGQ Market 해당 상품 페이지로 딥링크

### 4.4 월드 모델 뷰어 (`/world/{bookId}/{imageId}`)

#### 4.4.1 기능 개요

- WorldLabs Marvel API 연동으로 2D 삽화/사진 → 실시간 3D 공간 변환
- 1인칭 또는 3인칭 탐험 모드 선택
- 키보드(WASD) + 마우스 또는 터치 드래그로 이동
- 공간 내 등장인물 클릭 → AI Agent 대화 팝오버
- 공간 내 오브젝트 클릭 → 관련 책 내용 설명 팝업
- 스크린샷/짧은 클립 캡처 → 커뮤니티 공유 또는 2차 창작 소재로 저장
- 로딩 중 책 관련 퀴즈 또는 명언 표시 (대기 UX 개선)

#### 4.4.2 기술 연동

| 항목 | 내용 |
|------|------|
| 3D 엔진 | WorldLabs Marvel (또는 Google WorldModel) API |
| 입력 데이터 | 책 삽화 PNG/JPG + 장면 메타데이터 JSON |
| 출력 | 실시간 렌더링 WebGL 뷰 (iframe or 독립 URL) |
| AI 대화 연동 | 월드 공간 내 Claude API 기반 캐릭터 Agent |
| 접근성 | 3D 멀미 방지: FOV 제한 설정, 모션 감소 모드 |

## 5. AI Agent 시스템

### 5.1 Agent 유형

| Agent 유형 | 설명 | 학습 데이터 | 특징 |
|-----------|------|-----------|------|
| 주인공 Agent | 책의 주인공 캐릭터 | 책 전문 + 캐릭터 시트 | 1인칭 몰입 대화 |
| 저자 Agent | 실제 저자의 페르소나 | 저자 인터뷰·에세이 + 책 | 창작 의도 설명 |
| 조연 Agent | 주요 등장인물 | 책 전문 + 캐릭터 관계도 | 복수 운영 가능 |
| 월드 가이드 | 책 세계관 안내자 | 세계관 설정집 | 중립적 해설자 |
| 학습 튜터 | 주니어 도서 전용 | 책 내용 + 교육과정 연계 | 어린이 언어 최적화 |

### 5.2 Agent 설정 구조

#### 5.2.1 시스템 프롬프트 레이어

- **Layer 1** — 기본 안전 가이드라인 (플랫폼 공통, 수정 불가)
- **Layer 2** — 책 세계관/설정 데이터 (어드민 설정)
- **Layer 3** — 캐릭터 페르소나 + 말투 스타일 (어드민 설정)
- **Layer 4** — 대화 맥락 메모리 (사용자별 자동 관리)

#### 5.2.2 학습 데이터 소스

- 책 원문 텍스트 (전체 또는 허용 챕터)
- 캐릭터 시트: 이름, 나이, 성격, 말투, 관계, 비밀
- 어드민 업로드 추가 자료 (저자 노트, 인터뷰, 세계관 설정집)
- 금지 주제 설정: 어드민이 답변 금지 키워드/토픽 지정

### 5.3 대화 품질 관리

- 환각(hallucination) 방지: RAG 기반, 책 외 사실 주장 제한
- 부적절 응답 자동 감지 + 어드민 리포트
- 연령 맞춤: 주니어 도서는 어린이 언어 필터 필수 적용
- 사용자 피드백: 각 응답에 👍👎 평가 → 학습 개선 피드백

## 9. 상단 탭 네비게이션 (책 상세 페이지)

기존 10개 탭을 3개 대탭으로 묶고, 세부 유형은 하위 필터칩으로 처리한다. 콘텐츠가 없는 탭과 칩은 규칙에 따라 숨기거나 비활성 처리하여 플랫폼이 허전해 보이지 않도록 한다.

### 9.1 대탭 구성 (항상 최대 3개)

| 대탭 | 포함 콘텐츠 | 노출 조건 |
|------|-----------|---------|
| 창작 갤러리 | 2차 창작 전체 (스티커/음악/웹툰/영상/소설 등) | 항상 노출 (콘텐츠 0개여도) |
| 굿즈 | 한정판 굿즈 목록 | 굿즈 1개 이상 등록 시에만 노출 |
| 시리즈 | 같은 시리즈 도서 목록 | 시리즈 2권 이상 존재 시에만 노출 |

> **탭 노출 원칙**
> - 있는 것만 보여준다: 굿즈 0개 → 굿즈 탭 숨김 (비활성 X, 아예 미표시)
> - 시리즈 1권짜리 → 시리즈 탭 숨김
> - 창작 갤러리는 항상 노출 — 빈 상태도 "첫 창작자 되기" CTA로 활용

### 9.2 창작 갤러리 하위 필터칩

창작 갤러리 탭 선택 시 탭 아래에 가로 스크롤 필터칩 표시. 최대 10종이지만 화면에는 해당 책의 실제 콘텐츠 유형만 활성 칩으로 노출.

| 필터칩 | 포함 유형 | 활성 조건 | 비고 |
|--------|---------|---------|------|
| 전체 | 모든 2차 창작 | 항상 활성 | 기본 선택 |
| 스티커 | PNG/APNG 스티커 세트 | 1개 이상 승인 | - |
| 음악 | MP3/WAV 음원 | 1개 이상 승인 | 30초 미리듣기 |
| 웹툰 | 숏툰 (최대 10컷) | 1개 이상 승인 | - |
| 영상 | 쇼츠/웹드라마/동영상 | 1개 이상 승인 | 통합 필터 |
| 소설 | 숏북/챕터북 | 1개 이상 승인 | - |
| 기타 | AI Agent/프롬프트/크롬익스텐션 | 1개 이상 승인 | - |

### 9.3 빈 상태 (Empty State) 처리

| 상황 | 처리 방식 | CTA |
|------|---------|-----|
| 창작 갤러리 0개 | "아직 이 책의 창작물이 없어요. 첫 번째가 되어볼까요? ✨" | [창작물 올리기] |
| 특정 필터칩 0개 | 해당 칩 회색(비활성) 처리 + 클릭 불가 | 없음 |
| 굿즈 탭 (탭 숨김) | 탭 자체 미표시 | 없음 |
| 시리즈 탭 (탭 숨김) | 탭 자체 미표시 | 없음 |

각 탭/창작물 클릭 시 → 새 창으로 열림 (기존 독서 세션 유지)
